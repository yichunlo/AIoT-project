運作原理與實作過程：
我們將成品分為兩個部分：軟體介面與硬體層面。大架構來說，透過軟體介面的處理，設計師可以參考介面來預期硬體上的表現。而一旦設置完成啟動之後，該介面會將硬體資訊傳輸給控制硬體的主機，硬體再做出相對應的變化。以下是對於我們實作的部分加以說明：

1. 軟體介面
透過unity來實做出使用者介面，一共寫了四個C#的script，分別控制User Interface、Camera、inForm以及Input處理。User Interface有三種模式可以做選擇：Wave、Design和Game。Wave可以製造簡易的sin、cosine、tangent波形，並且可以調整波源、頻率與震幅。Design可以由鍵盤輸入控制球的滾動，此功能可融入於多種設計想法，故稱之為Design（圖一）。而Game則是設計遊戲，目前有簡易的小遊戲如貪吃蛇，可以藉由高低的變化來作為蛇運動時的表現，而標的物也會間隔一段時間上下震動以作提醒。未來展望是透過簡易的模式切換，設計師可以不用大費周章地操作或是寫大量的程式進行設計，而能夠透過簡易好操作的介面或是別人已經完成好的suit進行修改即可。

2. 硬體層面
我們最主要實際組裝的部分是inFORM，這是麻省理工學院七年前開發的一套Tangible user interface，為一種新型態的互動工具，我們將其加以延伸、應用。inFORM硬體系統可分為四個部分：
1. Top pins: 使用者實際接觸互動介面，以方形陣列呈現，底部有細小連桿連接。
2. Linkage: 中間連桿機構，如圖（二）所示。
3. Motor mechanism: 使用齒輪齒條機構，將馬達的旋轉運動轉為直線行程，如圖（三）所示
4. Ｃontrol System: 涵蓋了控制系統與動力來源，包括伺服馬達(sg90)、馬達驅動器（PCA9685）與控制板（Arduino)，可以讓頂部的Pin準確呈現的馬達矩陣系統。

硬體機構以raspberry pi進行控制，將資料傳輸給Arduino，最後達成硬體實際的變化與表現。 配合其他硬體物件，raspberry pi能夠達成物聯網的功效，增加inFORM的空間互動與進階的應用。
